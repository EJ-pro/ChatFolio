## ūüí° Project Overview

When joining a new open-source project or taking over a large legacy codebase, the cost of understanding the code is immense.

**ChatFolio** solves this problem. Simply enter a GitHub URL to:

- ūüďä Automatically extract **dependency graphs** across thousands of files
- ūü§Ė Instantly chat with an **AI code chatbot** powered by the extracted graph
- ūüďĚ Analyze the project's identity and let a multi-agent system automatically write a **high-quality README**

## ‚öôÔłŹ Core Architecture

### 1. AI Auto-Docs Multi-Agent Pipeline

ChatFolio's documentation engine is a 4-stage autonomous agent system built with LangGraph.

```mermaid
graph LR
    A(ūüď• GitHub URL) --> B

    subgraph Pipeline ["ūü§Ė Multi-Agent Workflow"]
        B[ūüĒć Analyzer\nIdentify project identity] -->|Archetype classification| C[ūüĒÄ Router\nDetermine optimal strategy]
        C --> D[‚úćÔłŹ Writer\nWrite Markdown draft]
        D --> E{ūüēĶÔłŹ Reviewer\nQuality review}
        E -->|Revision needed| D
        E -->|Approved| F
    end

    F(ūüďĄ Final README)
```

|   Agent      | Role                                                                                          |
| :----------: | --------------------------------------------------------------------------------------------- |
| **Analyzer** | Scans config files and language distribution to identify the project archetype (Backend / Mobile / ML, etc.) |
| **Router**   | Based on the identified archetype, decides which tech stack and sections to emphasize         |
| **Writer**   | Writes a Markdown draft using core file snippets from the dependency graph                    |
| **Reviewer** | Reviews Getting Started validity and technical accuracy, then runs a feedback loop to Writer  |

<br/>

Dynamically infers the project's identity starting from the URL collection stage.


### 3. Data Collection Pipeline (10 Steps)

From the moment a GitHub URL is entered until the RAG engine is ready, 10 pipeline stages execute.

```
 Step 01  ūüĒź Auth            Validate user PAT, check Rate Limit
    ‚Üď
 Step 02  ūüĒć Cache Check     Check latest commit SHA via GitHub API ‚Üí reuse cache or re-analyze
    ‚Üď
 Step 03  ūüď° Scan            Recursive directory traversal, filter .gitignore and binary files
    ‚Üď
 Step 04  ūüď• Fetch           Load large files without OOM via Python Generator streaming
    ‚Üď
 Step 05  ūüŹ≠ Parser Factory  Dynamically match optimal language parser by extension (Polyglot support)
    ‚Üď
 Step 06  ūüĆ≥ AST Parse       Extract and normalize classes, functions, and import statements per language
    ‚Üď
 Step 07  ūüíĺ Persist         Save parsed results + original code in a PostgreSQL transaction
    ‚Üď
 Step 08  ūüēł Graph Build     Map import paths to actual files via Resolver Factory, build NetworkX DiGraph
    ‚Üď
 Step 09  ūüďä Metrics         Identify core files via node degree calculation, convert to frontend JSON
    ‚Üď
 Step 10  ūü§Ė RAG Engine      Load analysis data into ChatFolioEngine, complete vector embeddings ‚Üí ready for chat
```

<br/>

### 4. Multi-Language Parser Support

A `Factory Pattern`-based parser router detects the file extension and automatically assigns the optimal analyzer.

## ūüöÄ Getting Started

### Prerequisites

- Docker & Docker Compose
- OpenAI API Key
- Groq API Key _(optional, free tier available)_

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/EJ-pro/ChatFolio.git
cd ChatFolio

# 2. Set up environment variables
cp .env.sample .env
# Open .env and enter your API keys

# 3. Run
docker-compose up --build
```

> ‚úÖ After starting, open your browser and go to `http://localhost`.
